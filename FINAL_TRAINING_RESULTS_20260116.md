## 四、关键修复说明

### 4.1 问题根源

**原始错误代码**（修复前）:
```python
# 错误：使用np.tile将所有帧的预测值设为相同
all_probs = np.concatenate(all_probs)  # (n_videos,)
n_frames = ds_params["n_frames"]
all_pred = np.tile(all_probs[:, None], (1, n_frames))  # ❌ 所有帧都是相同的值！
```

**问题**：
- `all_outputs`是一个列表，包含每个时间步的模型输出
- 但代码只使用了最后一个输出（`all_outputs[-1]`），然后将其复制到所有帧
- 这导致每个视频的所有帧都有相同的预测概率
- 在TTA计算中，由于所有帧相同，导致TTA值固定为5.0（视频总长度）

### 4.2 修复方案

**修复后代码**:
```python
# 正确：收集每个时间步的frame-level预测
all_pred_frame_list = []
for frame_idx in range(len(all_outputs)):
    frame_output = all_outputs[frame_idx]  # (B, 2)
    frame_probs = torch.softmax(frame_output, dim=-1)[:, 1]  # (B,)
    all_pred_frame_list.append(frame_probs.cpu().numpy())

# 转置并拼接得到 (n_videos, n_frames)
all_pred = np.array(all_pred_frame_list).T
```

**修复效果**：
- ✅ 每个时间步的预测都被正确收集
- ✅ 每个视频的帧级预测都是不同的
- ✅ TTA计算能够正确反映模型在不同时间点的预测能力
